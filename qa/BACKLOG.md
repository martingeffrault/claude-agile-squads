> This file is maintained by the QA team (@qa-manager).

## How to use this file

### For Dev Squads
1. Check this file at the beginning of each Sprint Planning
2. Critical (P1) bugs must be addressed first
3. Create stories in your backlog for assigned bugs
4. Update status when bug is fixed

### For QA Team
1. Add new bugs here after documenting them in `bug-reports/`
2. Prioritize by severity and impact
3. Assign to appropriate squad
4. Retest when status changes to "Fixed"
